================================================================================
INSTRUÇÕES PARA IMPLEMENTAR A PÁGINA INTERMEDIÁRIA DE DOWNLOAD COM ANÚNCIOS (ADS)
================================================================================

Este documento serve como um guia/passo a passo detalhado para o desenvolvedor de IA
implementar o sistema de intermediação de downloads com monetização de anúncios (PopAds + Adsterra)
no futuro.

---

### OBJETIVO
Redirecionar o clique nos botões "Download Source" para uma página interna do próprio site
(ex: `download.html`), onde o usuário passará por etapas de interação e visualização de anúncios
antes de ter o link definitivo liberado, maximizando os ganhos com CPM e cliques sem risco de banimento.

---

### FLUXO DO USUÁRIO (UX)

1. **Clique no Modal:** 
   O usuário abre um vídeo no modal e clica em um link "Download Source X".
   O site abre a página `download.html?dest=<URL_CODIFICADA>` em uma nova aba.

2. **Tela 1 (Desbloqueio):**
   Exibe um aviso: "LIBERE SEU DOWNLOAD - Abra 1 anúncio para continuar."
   Botão: "ABRIR ANÚNCIO E CONTINUAR".
   Ao clicar neste botão:
   - Roda um `window.open(link_popads, '_blank')` (gera receita com PopAds).
   - Oculta a Tela 1 e exibe a Tela 2 na mesma aba.

3. **Tela 2 (Aguardando / Banners):**
   Exibe uma mensagem: "AGUARDE PARA LIBERAR O CONTEÚDO" com um contador circular progressivo de 5 a 10 segundos.
   Abaixo do contador, exibe blocos de anúncios (Banners do Adsterra ou Social Bar).
   Quando o cronômetro chega a zero, oculta a Tela 2 e exibe a Tela 3.

4. **Tela 3 (Liberado):**
   Exibe: "Download Liberado! Seu acesso foi confirmado."
   Botão: "DOWNLOAD" (aponta para o link original decodificado).

---

### ETAPA 1: CRIAÇÃO DO ARQUIVO `download.html`

Crie um novo arquivo chamado `download.html` na raiz do projeto. Ele deve seguir a mesma estética
dark e elegante do site (paleta de cores escuras, tons de cinza/azul e destaques em rosa `#ec4899` ou verde `#10b981`).

#### Estrutura Básica do Código para `download.html`:

```html
<!DOCTYPE html>
<html lang="pt-BR">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Aguarde seu Download — RIPNSFW</title>
    <style>
        body {

            background: #020617;
            color: #e2e8f0;
            font-family: system-ui, -apple-system, sans-serif;
            display: flex;
            align-items: center;
            justify-content: center;
            height: 100vh;
        }
        .container {
            width: 90%;
            max-width: 480px;
            background: #0b1223;
            border: 1px solid rgba(255, 255, 255, 0.07);
            border-radius: 1.25rem;
            padding: 2rem;
            text-align: center;
        }
        .btn {
            background: #10b981;
            color: white;
            border: none;
            padding: 0.85rem 1.5rem;
            border-radius: 0.75rem;
            font-weight: bold;
            cursor: pointer;
            width: 100%;
            font-size: 1rem;
            margin-top: 1.5rem;
        }
        .timer-ring {
            /* Estilo para animação do cronômetro circular */
        }
        .ad-slot {
            margin: 1.5rem 0;
            min-height: 100px;
            background: rgba(255,255,255,0.02);
            border-radius: 0.5rem;
        }
    </style>
</head>
<body>
    <div class="container">
        <!-- TELA 1 -->
        <div id="step-1">
            <h2>LIBERE SEU DOWNLOAD</h2>
            <p>Clique abaixo para abrir 1 anúncio e continuar.</p>
            <button class="btn" onclick="startVerification()">ABRIR ANÚNCIO E CONTINUAR</button>
        </div>

        <!-- TELA 2 -->
        <div id="step-2" style="display:none">
            <h2>Aguarde para liberar o conteúdo...</h2>
            <div id="countdown">10</div>
            <!-- Cole aqui os códigos de anúncio do Adsterra (Banners/Native) -->
            <div class="ad-slot">
                <!-- Script do Adsterra -->
            </div>
        </div>

        <!-- TELA 3 -->
        <div id="step-3" style="display:none">
            <h2>Download Liberado!</h2>
            <button class="btn" id="download-btn">DOWNLOAD</button>
        </div>
    </div>

    <script>
        // Função para ler a URL de destino codificada em Base64
        function getDestUrl() {
            const params = new URLSearchParams(window.location.search);
            const encoded = params.get('dest');
            try {
                return atob(encoded); // Decodifica Base64
            } catch(e) {
                return null;
            }
        }

        function startVerification() {
            const dest = getDestUrl();
            if (!dest) return alert('Link inválido!');

            // 1. Abre o pop-under (PopAds ou Adsterra Popunder)
            window.open('URL_DO_ANUNCIO_POPADS', '_blank');

            // 2. Passa para a tela de espera
            document.getElementById('step-1').style.display = 'none';
            document.getElementById('step-2').style.display = 'block';

            // 3. Cronômetro
            let seconds = 10;
            const timer = setInterval(() => {
                seconds--;
                document.getElementById('countdown').textContent = seconds;
                if (seconds <= 0) {
                    clearInterval(timer);
                    showDownloadLink(dest);
                }
            }, 1000);
        }

        function showDownloadLink(dest) {
            document.getElementById('step-2').style.display = 'none';
            document.getElementById('step-3').style.display = 'block';
            document.getElementById('download-btn').onclick = () => {
                window.location.href = dest;
            };
        }
    </script>
</body>
</html>
```

### ETAPA 2: ALTERAÇÕES NO ARQUIVO `index.html` DO PROJETO PRINCIPAL

No arquivo `index.html`, devemos alterar a renderização dos botões da coluna I na função `openVideo()`.

#### Código a ser modificado:
Na função `openVideo()`, onde são gerados os links `dlLinks.appendChild(a)` da coluna I, altere o link para passar pelo `download.html` apenas na primeira parte (ou link único). As partes adicionais (Part 2, Part 3, etc.) abrem direto no Bunkr sem passar pela página de anúncio, para melhorar a experiência do usuário.

*Exemplo de como deve ser a alteração:*

```javascript
            if (colILinks.length > 0) {
                colILinks.forEach((link, i) => {
                    const a = document.createElement('a');
                    const realLink = link.startsWith('www.') ? 'https://' + link : link;
                    
                    if (i === 0) {
                        // Apenas a Part 1 (ou link único) passa pelo anúncio intermediário
                        const encodedLink = btoa(realLink);
                        a.href = '/download.html?dest=' + encodedLink;
                    } else {
                        // As demais partes (Part 2, 3...) abrem direto no link final sem propaganda intermediária
                        a.href = realLink;
                    }
                    
                    a.target = '_blank';
                    a.rel = 'noopener noreferrer';
                    a.className = 'dl-link ' + (i === 0 ? 'primary' : 'secondary');
                    if (colILinks.length > 1) {
                        a.innerHTML = 'Download Part ' + (i + 1) + ' Source ' + EXT_ICON;
                    } else {
                        a.innerHTML = 'Download Source 1 ' + EXT_ICON;
                    }
                    dlLinks.appendChild(a);
                });
            }
```
